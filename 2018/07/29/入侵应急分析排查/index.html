入侵应急分析排查

发表于 2018-07-29 | 分类于 安全审计
应用服务器入侵应急分析手工排查

事件类型，排查步骤

事件类型

  • Web入侵：挂马、篡改、Webshell
  • 系统入侵：系统异常、登录爆破、系统漏洞
  • 病毒木马：远控、后门、垃圾软件
  • 信息泄漏：脱库、数据库注入、弱口令登录
  • 网络流量：频繁发包、批量请求、DDOS

步骤

攻击类型定位 –> 确定时间范围 –> 进程分析 –> 文件分析 –> 系统分析 –> 日志分析 –> 关联分析 –> 推理总结
注：
1、时间范围界定，关键操作时间点串联
2、Web入侵shell定位

系统信息排查

Windows

  • 查看系统变量的设置：set
  • Windows 计划任务：at / 计算机管理 -> 任务计划程序
  • Windows 帐号信息：net users / 计算机管理 -> 本地用户和组
  • 查看开机启动项：msconfig
  • 检索注册表，SAM文件及远控类软件
  • systeminfo信息，系统版本以及补丁信息

Linux

  • 查看命令历史(.bash_history)
    1、下载文件(wget)
    2、尝试远程(ssh、scp)
    3、打包数据(tar、zip)
    4、对系统进行配置(vim、export、echo $PATH)
  • 查看用户相关信息
    1、useradd、userdel
    2、/etc/passwd
    查看UID为0的帐号
cat /etc/passwd | awk -F':' '{if($3==0)print $0}'
查看能登录的帐号
cat /etc/passwd | grep /bin/bash
  • 查看任务计划
    1、列出任务计划：rontab -l
    2、排查任务计划文件：ls /etc/cron*
  • 查看开机加载项
    1、rc.local文件(/etc/init.d[rc.local] - ls -alt /etc/init.d/
    2、chkconfig/sysv-rc-conf
  • 查看用户登录信息
    1、lastlog：系统中所有用户最近一次登录信息
    2、lastb：用户错误的登录列表
    3、last：用户最近的登录信息
    4、ssh连接日志(authpriv.*)：auth.log[secure]

进程信息分析

Windows

  • 查看目前的网络连接，定位已经建立的连接(ESTABLISHED)
    netstat -ano | findstr ESTABLISHED
  • 根据PID进行进程定位
    tasklist [\svc]| findstr PID
结束可疑的进程
tskill PID|EXENAME

Linux

  • 网络连接，PORT、IP、PID
    netstat -anltp
  • 进程分析
    ps aux | grep PID
  • 端口分析
    lsof -i:PORT
  • 查看系统命令是否被修改
    ls -alt /usr/bin\[sbin] | head -n 10

文件信息分析

Windows

  • 文件属性(时间)
  • 浏览器记录(浏览历史，下载文件，cookie)
  • 最近访问文件
    *\User\Recent
%UserProfile%\Recent
  • 文件夹搜索
    1、支持正则匹配
    2、支持类型匹配(类型:.csv)
    3、支持时间匹配
  • command：findstr
    查看日志中访问上传文件夹的记录
findstr /s /m /I "UploadFiles" *.log
查找含有某关键字的文件
findstr /s /m /I "str" *.aspx

Linux

  • 敏感文件
    1、/tmp/、/etc/init.d[passwd、shadow]、/usr/bin[sbin]
    2、按时间排序显示：ls -alt
    3、查看文件详细信息：stat FILE

  • 新增文件分析
    1、24小时内被修改的jsp文件

    find ./ -mtime 0 -name "\*.jsp"

    2、查找2天内新增的文件

    find / -ctime -2

    3、特殊权限文件

    find / *.jsp -perm 4777

    4、文件目录下关键字搜索

    grep -r STRING

  • 涉及文件分析的命令集

    find grep ls stat

后门排查

Windows

  • PC Hunter
    http://www.xuetr.com/
1.进程、线程、进程模块、进程窗口、进程内存信息查看，杀进程、杀线程、卸载模块等功能
2.内核驱动模块查看，支持内核驱动模块的内存拷贝
3.SSDT、Shadow SSDT、FSD、KBD、TCPIP、Classpnp、Atapi、Acpi、SCSI、IDT、GDT信息查看，并能检测和恢复ssdt hook和inline hook
4.CreateProcess、CreateThread、LoadImage、CmpCallback、BugCheckCallback、Shutdown、Lego等Notify Routine信息查看，并支持对这些Notify Routine的删除
5.端口信息查看，目前不支持2000系统
6.查看消息钩子
7.内核模块的iat、eat、inline hook、patches检测和恢复
8.磁盘、卷、键盘、网络层等过滤驱动检测，并支持删除
9.注册表编辑
10.进程iat、eat、inline hook、patches检测和恢复
11.文件系统查看，支持基本的文件操作
12.查看（编辑）IE插件、SPI、启动项、服务、Host文件、映像劫持、文件关联、系统防火墙规则、IME
13.ObjectType Hook检测和恢复
14.DPC定时器检测和删除
15.MBR Rootkit检测和修复
16.内核对象劫持检测
17.WorkerThread枚举
18.Ndis中一些回调信息枚举
19.硬件调试寄存器、调试相关API检测
20.枚举SFilter/Fltmgr的回调
21.系统用户名检测
  • webshell
    应用目录拷贝，进行恶意函数匹配
工具：河马、D盾...

Linux

  • 工具
    1、chkrootkit
    chkrootkit | grep INFECTED
    2、rkhunter
    rkhunter --update
rkhunter --check --sk
  • webshell查找
    Webshell的排查可以通过文件、流量、日志 三种方式进行分析,基于文件的命名特征和内容特征,相对操作性较高,在入侵后应急过程中频率也比较高
    find ./ -name "*.php" | xargs egrep "assert|phpspy|c99sh|milw0rm|eval|\(gunerpress|assert|phpspy|c99sh|milw0rm|eval|\(gunerpress||\(\$\_\POST\[|eval \(str_rot13|\.chr\(|\$\{\"\_P|eval\(\$\_R|file_put_content|base64_decode"

综上所述,通过chkrootkit 、rkhunter、RPM check、Webshell Check 等手段得出以下应对措施:

  1. 根据进程、连接等信息关联的程序,查看木马活动信息。
  2. 假如系统的命令(例如netstat ls 等)被替换,为了进一步排查,需要下载一新的或者从其他未感染的主机拷贝新的命令。
  3. 发现可疑可执行的木马文件,不要急于删除,先打包备份一份。
  4. 发现可疑的文本木马文件,使用文本工具对其内容进行分析,包括回连IP地址、加密方式、关键字(以便扩大整个目录的文件特征提取)
