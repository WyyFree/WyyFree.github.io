Linux日志分析(awk,grep,wc...)，Windows日是分析(LogParser)

Linux

日志文件(/var/log)

  • 日志文件配置(/etc/rsyslog.conf[syslog.conf])：用于控制/var/log下的日志文件记录的内容
    配置文件说明
    参数格式：类型.级别[;类型.级别]    动作
    类型：
    auth    认证系统，即询问用户名和口令(syslog)
authpriv   权限使用相关信息(auth.log/secure)
user    本地用户应用程序的syslog信息
cron    系统定时系统执行定时任务时发出的信息
daemon  某些系统的守护程序的syslog,如由in.ftpd产生的log
kern    内核的syslog信息
lpr     打印机的syslog信息
mail    邮件系统的syslog信息
mark    定时发送消息的时标程序
news    新闻系统的syslog信息
uucp    uucp子系统的syslog信息
local0..7 种本地类型的syslog信息,这些信息可以又用户来定义
*       代表以上各种设备
    级别
    emerg   紧急，处于Panic状态。通常应广播到所有用户； 
alert   告警，当前状态必须立即进行纠正。例如，系统数据库崩溃； 
crit    关键状态的警告。例如，硬件故障； 
err     其它错误； 
warning 警告； 
notice  注意；非错误状态的报告，但应特别处理； 
info    通报信息； 
debug   调试程序时的信息； 
none    通常调试程序时用，指示带有none级别的类型产生的信息无需送出。如*.debug;mail.none表示调试时除邮件信息外其它信息都送出。
    动作
    /filename   日志文件。由绝对路径指出的文件名，此文件必须事先建立； 
@host       远程主机； @符号后面可以是ip,也可以是域名，默认在/etc/hosts文件下loghost这个别名已经指定给了本机。
user1,user2 指定用户。如果指定用户已登录，那么他们将收到信息； 
*           所有用户。所有已登录的用户都将收到信息。
  • 其他日志
    /var/log/btmp 记录错误的登录尝试(auth/secure)。
    /var/log/wtmp 记录所有登入登出。
        last -f wtmp
    /var/log/lastlog 记录每个用户最后登入信息。
        command：last
    /var/run/utmp 记录现在登入的用户。
        command：w/who

Windows

事件查看器

打开事件管理器(开始—管理工具—事件查看/开始运行eventvwr)
Windows server 2008 系列参考 event ID :

4624 - 帐户已成功登录 
4625 - 帐户登录失败 
4648 - 试图使用明确的凭证登录(例如远程桌面）

分析方法

  1. 主要分析安全日志,可以借助自带的筛选功能
  2. 可以把日志导出为文本格式,然后使用文本编译器打开,使用正则模式去匹配或者过滤远程登录过的IP地址,在界定事件日期范围的基础
    正则匹配IP
((?:(?:25[0-5]|2[0-4]\d|((1\d{2})|([1-9]?\d))).){3}(?:25[0-5]|2[0-4]\d|((1\d{2})|([1-9]?\d))))
  3. 日志分析工具Log Parser

中间件

  • 常用中间件类型：
    apache
    日志文件配置位置：apache.conf[httpd.conf]
    nginx
    日志文件配置位置：nginx.conf
    tomcat
    日志文件配置位置：conf/server.xml
    IIS
    日志文件配置位置：日志 -> 目录 -> 浏览

  • 无论任何web服务器其实日志需要关注的东西是一致的,即access_log和error_log。

  • 排查方法：
    1、确定攻击时间范围
    2、确定攻击载荷
    3、确定攻击IP

工具

Linux

  • cat：查看文件内容
    cat file1 file2
cat *.log
cat access*
  • more|head|tail|wc
    cat file | more[less] #分页显示,less可在显示也搜索(/?)
cat file | head[tail] -n LINE #输出前[末尾]n行
cat file | tail -f #实时显示刷新内容
cat file | wc -l #统计行数
  • grep
    cat file | grep "STR"
常用参数：
    -r STR：当前目录文件内容关键字搜索
    -v STR：排除包含STR的行
    -i STR：忽略大小写
    -n：在前面显示行数
  • awk：分离行匹配输出
    输出，$0：整行，-F：指定分割的符号(默认空格)
awk -F 'str' '{print $field}'
去重
'!a[$filed]++{print $field}'
判断
'{if($field==continue)print $0}'
切割
'{split($field,value,str);print value[i]}'
  • 用例
    cat *.log | awk '!a[$3]++{print $3}'
cat file | awk '{split($1,a,"-");if(a[3]==20)print $0}' | more

Windows

  • Logparser
    1、格式
    logparser -i:evt -o:[datagrid] "SQL"
参数：
-i：输入源
-o：输出源(net[默认]、datagrid、csv)
SQL：查询语句
    2、用例
    Logparser.exe –i:EVT –o:DATAGRID “SELECT * FROM E:\logparser\xx.evtx”
LogParser.exe -i:EVT "SELECT TimeGenerated,EXTRACT\_TOKEN(Strings,0,'|') AS USERNAME,EXTRACT\_TOKEN(Strings,2,'|') AS SERVICE\_NAME,EXTRACT\_TOKEN(Strings,5,'|') AS Client\_IP FROM 'e:\logparser\xx.evtx' WHERE EventID=675 AND EXTRACT\_TOKEN(Strings,5,'|')='x.x.x.x'"
    3、用logparser导出成csv格式(数据在同一行)
    Logparser.exe –i:EVT –o:csv “SELECT * FROM E:\logparser\xx.evtx” > new.csv
如果在Linux上打开会出现乱码，需要word重新打开保存
